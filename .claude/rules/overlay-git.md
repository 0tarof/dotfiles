# overlayディレクトリのGit操作に関するルール

## 重要なルール

**このリポジトリからoverlayディレクトリの変更をコミット・プッシュしないこと。**

`overlay/`ディレクトリは環境固有の設定を管理しており、独自のGitリポジトリを持っています。`overlay/`内のファイルへの変更は、overlayディレクトリ内で別途コミット・プッシュする必要があります。

## 正しいワークフロー

`overlay/`内のファイルに変更を加えた場合：

1. overlayディレクトリに移動する：
   ```bash
   cd overlay
   ```

2. そこでコミット・プッシュする：
   ```bash
   git add <変更したファイル>
   git commit -m "コミットメッセージ"
   git push
   ```

3. 作業完了後、親ディレクトリに戻る：
   ```bash
   cd ..
   # または
   cd /Users/a14993/projects/github.com/0tarof/dotfiles
   ```

## なぜこれが重要か

- `overlay/`には環境固有の設定（例：業務用PCの設定）が含まれる
- 独立したGitリポジトリとして管理されている
- 親リポジトリからoverlayの変更をコミットすると、コンフリクトや不正確なバージョン管理が発生する

## 代わりに何をすべきか

- このリポジトリからは基本設定のみをコミット・プッシュする
- overlayの変更は分離して、独自のリポジトリで管理する
- 親リポジトリのルートから`git add overlay/`を実行しない
