# GitHub Activity スキル

今日のGitHubアクティビティを取得し、わかりやすくまとめるスキル。

**IMPORTANT: 日本語でユーザーとコミュニケーションを取ること。**

## ワークフロー

### 1. データ取得

スクリプトを実行して今日のアクティビティをJSON形式で取得する：

```bash
${CLAUDE_SKILL_DIR}/scripts/fetch_activity.sh
```

### 2. プレゼンテーション

スクリプトの出力JSONを以下のテンプレートに従って整形する。件数が0のセクションは省略する。

```markdown
## YYYY-MM-DD GitHub アクティビティまとめ

### PR マージ (N件)
**repo-name** (M件)
- `#123` タイトル
- `#456` タイトル

### PR 作成 (N件)
**repo-name** (M件)
- `#123` タイトル

### PR レビュー Approve (N件)
**repo-name** (M件)
- `#123` タイトル

### PR レビュー Changes Requested (N件)
**repo-name** (M件)
- `#123` タイトル

### Issue 作成 (N件)
**repo-name** (M件)
- `#123` タイトル

### Issue クローズ (N件)
**repo-name** (M件)
- `#123` タイトル

### その他
- コメントしたIssue/PR一覧
- pushしたリポジトリ一覧

---

### ざっくりまとめ

大きく **N つの軸** で動いてた：

1. **テーマ名** — 関連する作業の説明
2. **テーマ名** — 関連する作業の説明
...
```

### プレゼンテーションルール

1. **リポジトリでグルーピング**: 各セクション内で同じリポジトリのアイテムをまとめる
2. **リポジトリ名の表示**: `owner/name` の `name` 部分のみ太字で表示する
3. **0件セクション省略**: 該当なしのセクションは出力しない
4. **pushes**: リポジトリ名と ref を箇条書きで表示。同一リポジトリ・同一refへの複数pushは重複排除する
5. **comments**: コメントしたIssue/PRをリポジトリでグルーピングして表示。同一Issue/PRへの複数コメントは重複排除する

### ざっくりまとめの作り方

- 個々のPR/Issueのタイトルからテーマ（機能開発、バグ修正、インフラ改善、コード品質など）を推測してグルーピングする
- 単にカテゴリを列挙するのではなく、「何に取り組んでいたか」が一目で分かるようにする
- テーマ名は具体的に（例: 「認証基盤のリファクタ」「検索機能の改善」）
- 1つのテーマに関連するPR/Issueが複数ある場合、それらをまとめて説明する
