# PR作成時のテンプレート参照ルール

PRを作成するときは、リポジトリ内のPRテンプレートを必ず確認し、その構成に沿って本文を書く。

## 確認する場所

以下を順に確認し、存在すれば最初に見つかったものを使う。

1. `.github/PULL_REQUEST_TEMPLATE.md`
2. `.github/pull_request_template.md`
3. `.github/PULL_REQUEST_TEMPLATE/` 配下の `.md` ファイル（複数ある場合は変更内容に最も近いものを選ぶ）
4. `docs/pull_request_template.md`

## 運用

- テンプレートのセクション構成（見出し・チェックリスト・コメント）はそのまま踏襲する。
- `<!-- ... -->` のHTMLコメントは指示として読み、本文には残さない（テンプレ側で「消すな」と明記されている場合のみ残す）。
- チェックリストは実際に実施した項目だけチェックを入れ、未実施の項目は未チェックのまま残す。
- テンプレートにない情報を足すのは構わないが、既存セクションは勝手に省略・改変しない。
- テンプレートが見つからない場合のみ、従来通りの `## Summary` / `## Test plan` 形式でPR本文を書く。

## 実装メモ

- `gh pr create --body` に渡す本文は、テンプレートを読み込んだ上で変更内容を埋めて組み立てる。
- テンプレートが長い場合でも省略せず、全セクションを本文に含める。
